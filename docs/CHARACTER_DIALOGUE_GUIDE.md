# 마실펫 캐릭터·대사 가이드

## 설계 원칙

- 내부 `templateId`와 에셋 키는 유지하고 표시 이름만 개편해 기존 저장 데이터와 이미지 연결을 보호한다.
- 캐릭터는 지역명을 반복하는 대신 관찰 방식, 문장 길이, 비유 소재, 반응 속도로 구분한다.
- 방언은 캐릭터를 희화화할 수 있어 과장하지 않는다. 지역성은 자연·생활·문화 소재로 드러낸다.
- 대사에는 `보상`, `EXP`, `친밀도` 같은 시스템 용어를 넣지 않는다.
- 사용자를 재촉하거나 죄책감을 주지 않는다. 상태가 낮을 때는 필요를 솔직히 말하되 부드럽게 제안한다.

## 지원 상황

모든 캐릭터는 아래 상황에 빠짐없이 대응한다.

- 일상: `default` 2종, `morning`, `afternoon`, `evening`
- 필요 상태: `hungry`, `dirty`, `tired`
- 돌봄 결과: `fed`, `played`, `cleaned`, `resting`
- 관계 변화: `close`, `evolved`
- 방문 기억: `nature`, `food`, `festival`, `culture`, `history`, `shopping`, `other`

## 대화 선택 순서

1. 배고픔·청결·활력 중 하나가 45 미만이면 가장 낮은 상태의 대사를 우선한다.
2. 상태가 안정적이면 첫 대화에서 최근 방문 카테고리를 회상한다.
3. 이후 대화는 시간대, 기본 대사, 친밀 대사, 진화 대사를 순환한다.
4. 말풍선은 시스템 알림을 캐릭터의 말처럼 읽지 않는다. 직전 메시지가 실제 대사일 때만 유지하고, 아니면 현재 상태와 시간대에 맞는 대사를 고른다.
5. 먹기·놀기·씻기·쉬기 직후에는 행동 결과에 맞는 캐릭터 고유 반응을 바로 보여준다.

## 운영 메모

- 표시 이름을 바꿔도 내부 ID는 바꾸지 않는다.
- 기존 사용자에게 저장된 옛 이름은 불러올 때 현재 템플릿 이름으로 자동 보정한다.
- 서버 시드는 장소 방문용 8개 트리거를 저장하며, 시간·돌봄·관계 대사는 앱의 정적 카탈로그에서 즉시 선택한다.
- 새 캐릭터를 추가할 때는 이름 중복, 전체 상황 트리거, 시스템 용어 포함 여부를 자동 테스트로 확인한다.
